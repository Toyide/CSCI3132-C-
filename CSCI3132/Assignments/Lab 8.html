Concepts, Generic programing

Traditional programming.

#include <iostream>
#include <vector>
#include <string>

int main()
{  
    // Traditional code:
    double sqrt(double d); // C++84: accept any d that is a double
    double d = 7;
    double d2 = sqrt(d); // fine: d is a double
    std::vector<std::string> vs = { "Good", "old", "templates" };
    double d3 = sqrt(vs); // error: vs is not a double
}

Generic code was first used in 1990s.

template<class T> void sort(T& c) // C++98: accept a c of any type T
{
// code for sorting (depending on various properties of T,
// such as having [] and a value type with <
}
    vector<string> vs = { "Good", "old", "templates" };
    sort(vs); // fine: vs happens to have all the syntactic properties required by sort  

    double d = 7;
    sort(d); // error: d doesn’t have a [] operator

Problems:

  • No precise error while handling doubles
  • The template< typename T> is hard to work with.
  • The requirement of sort is hidden in its function body.

Concepts allow us to specify the requirements of a template.

    // Generic code using a concept (Sortable):
    void sort(Sortable& c); // Concepts: accept any c that is Sortable

    vector<string> vs = { "Hello", "new", "World" };
    sort(vs); // fine: vs is a Sortable container

    double d = 7;
    sort(d); // error: d is not Sortable (double does not provide [], etc.)

What is Sortable:

It basically is a random access sequence with elements that can be compared using the < operator.

Using concepts we can specify a template type requirement like:

  • An iterator Iterator< T>
  • A number Number< T>

In concepts, C < T> means C is a concept and T is an expression where it is true if T meets all the requirements of C and false otherwise.

we write concepts like:

template< Sequence Seq>
void algo(Seq& s);

is the same as

template< typename Seq>
    requries Sequence<Seq>
void algo(Seq& s);

Therefore, concepts are declared as follows:

template<C T>

or

template< typename T>
    requires C<T>
